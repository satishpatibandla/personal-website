# Satish Patibandla - Personal Website

A static, CMS-controlled personal website deployed on Netlify. Content is managed through Decap CMS at `/admin/`, committed to GitHub through Netlify Git Gateway, and rebuilt by Netlify on each publish.

## How Publishing Works

1. Log in at `https://satishpatibandla.netlify.app/admin/`.
2. Create or edit content in Decap CMS.
3. Click **Publish**.
4. Decap CMS commits the change to GitHub `main`.
5. Netlify runs `npm run build`, generates the JSON indexes, and deploys the site.

You should not need to manually deploy for normal blog or resource updates. If GitHub receives the CMS commit but Netlify does not deploy, check the Netlify deploy connection or trigger **Clear cache and deploy site** once.

## CMS Collections

- **Site Settings**: hero, profile, about content, skills, stats, resume, social links, contact copy, and homepage limits.
- **Blog Posts**: individual Markdown posts in `content/blog/*.md`.
- **Resources**: individual Markdown resource entries in `content/resources/items/*.md`.

Generated browser indexes:

- `content/blog/posts.json`
- `content/resources/resources.json`

These are generated by `scripts/build-content.mjs`.

## Local Commands

```bash
npm run build
```

This regenerates the post and resource indexes from CMS Markdown files.

## Project Structure

```text
admin/
  config.yml              # Decap CMS configuration
  index.html              # Admin entrypoint
content/
  blog/                   # Blog Markdown source + generated posts.json
  resources/items/        # Resource Markdown source
  resources/resources.json# Generated resources index
  site/settings.json      # CMS-managed site profile/settings
css/style.css             # Shared visual system
js/main.js                # Home page, settings, navigation, contact form
js/blog.js                # Blog list rendering
js/post.js                # Blog detail rendering
js/resources.js           # Resource list rendering
scripts/build-content.mjs # Content index generator
```

## Netlify Setup

Required settings:

- Identity enabled
- Registration set to **Invite only**
- Git Gateway enabled
- GitHub OAuth provider enabled for Identity login
- Build command: `npm run build`
- Publish directory: `.`

## Notes

- The homepage no longer uses hardcoded blog/resource cards. It reads the generated CMS indexes.
- Blog detail pages render the Markdown body stored in each post.
- Resources can link to an uploaded file or an external URL.
